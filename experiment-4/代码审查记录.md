## 审查发现的问题

### 问题 1：新创建的 ReviewSchedule 对象属性为 None

| 属性 | 内容 |
|------|------|
| **严重程度** | 🔴 高 |
| **文件** | `main.py:148` |
| **问题描述** | 创建新的 ReviewSchedule 对象时未传递默认参数，SQLAlchemy 的 `default` 参数是数据库级别的，在 flush 前 Python 对象属性为 None，导致 SM-2 计算时报错 `TypeError: unsupported operand type(s) for +: 'NoneType' and 'float'` |
| **修复方案** | 在构造函数中显式传递默认值：`ReviewSchedule(question_id=..., easiness_factor=2.5, interval=0, repetitions=0)` |
| **状态** | ✅ 已修复 |

### 问题 2：重复数据库初始化导致 seed 数据重复
