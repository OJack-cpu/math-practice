## 什么是 OpenCode？

OpenCode 是一个**开源 AI 编程助手**，运行在终端中。它能够理解你的代码库，帮助你完成代码编写、重构、调试、文档编写等任务。

核心特性：

- **代码库感知** — 自动理解项目结构和代码上下文
- **多会话** — 同时启动多个 agent 并行工作
- **免费模型** — 内置免费模型，开箱即用
- **隐私优先** — 你的代码和数据不会被存储
- **LSP 支持** — 自动加载语言服务器为 LLM 提供上下文

## 安装 OpenCode

```bash
npm install -g opencode-ai
```

验证安装：

```bash
opencode --version
```

### 开箱即用

OpenCode **内置免费模型**，安装后无需额外配置 LLM 提供商即可直接使用。进入项目目录，运行 `opencode` 即可开始。

## TUI 界面速览

启动 `opencode` 后进入终端交互界面（TUI）：

```
$ opencode
```

界面主要区域：

- **输入区** — 底部输入你的消息
- **对话区** — 显示你和 AI 的对话历史
- **状态栏** — 显示当前模式（Build/Plan）、模型等信息

### 基本对话

直接输入你的问题：

```
请给我一个项目结构的快速总结。
```

### 引用文件

使用 `@` 进行模糊搜索并引用文件：

```
@src/main.ts 中的认证逻辑是如何处理的？
```

引用后该文件内容会自动加入对话上下文。

### 运行 Shell 命令

以 `!` 开头可以直接执行 shell 命令，输出结果会作为上下文发给 AI：

```
!ls -la
```

### Plan / Build 模式切换

按 **Tab 键** 切换两种模式：

- **Build 模式**（默认）— AI 可以直接修改你的代码
- **Plan 模式** — AI 只出计划，不动代码

建议工作流：先用 Plan 模式讨论方案，确认后再切回 Build 模式让 AI 执行。

## 核心指令详解

### `/init` — 初始化项目

这是使用 OpenCode 的**第一步**。

**作用：** 让 OpenCode 分析你的项目，生成 `AGENTS.md` 文件。

**分析的内容：**

- 项目使用的编程语言和框架
- 项目目录结构
- 构建脚本和配置文件（`package.json`、`Cargo.toml`、`Makefile` 等）
- 已有的 lint / 测试配置
- 现有的规则文件（如 `.cursor/rules/`、`CLAUDE.md`）

**生成的 `AGENTS.md` 是什么？**

`AGENTS.md` 是项目的**规则说明书**，告诉 AI 这个项目怎么建、怎么测、代码风格是什么。例如：

```markdown
# My Project

## 技术栈
- TypeScript + React
- Vite 构建
- Vitest 测试

## 构建与测试
- 构建: `npm run build`
- 测试: `npm test`
- Lint: `npm run lint`

## 目录结构
- `src/components/` — UI 组件
- `src/hooks/` — 自定义 Hooks
- `src/utils/` — 工具函数
```

**什么时候读取 `AGENTS.md`？**

每次 **OpenCode 启动时**自动读取。启动后，AI 会在上下文里看到这份规则，让它了解你的项目规范。也就是说，你只需要生成一次，后续每次使用都会生效。

**位置：** 放在项目的根目录，建议提交到 Git，团队共享。

**手动创建：** 你也可以手动写 `AGENTS.md`，格式同上。

---

### `/connect` — 连接 LLM 提供商

如果你不想用内置免费模型，想用自己的 API 密钥：

```
/connect
```

从列表中选择提供商（如 Anthropic、OpenAI、GitHub Copilot 等），按提示完成认证。

认证信息保存在 `~/.local/share/opencode/auth.json`。

### `/models` — 选择模型

列出当前可用的所有模型：

```
/models
```

如果你刚通过 `/connect` 添加了新提供商，或者想换一个模型，用这个命令选择。

快捷键：`ctrl+x m`

### `/new` — 新建会话

清空当前对话，开启一个全新的会话：

```
/new
```

快捷键：`ctrl+x n`

### `/sessions` — 切换会话

查看所有历史会话，切换到之前的某个会话继续工作：

```
/sessions
```

快捷键：`ctrl+x l`

### `/undo` — 撤销

撤销上一次 AI 的操作（包括代码修改和对话）：

```
/undo
```

可以多次执行，一步步往回撤。

> 注意：依赖 Git，项目需要是 Git 仓库。

快捷键：`ctrl+x u`

### `/redo` — 重做

恢复被 `/undo` 撤销的操作：

```
/redo
```

快捷键：`ctrl+x r`

### `/compact` — 压缩上下文

当对话太长、接近 token 上限时，压缩当前会话上下文，保留关键信息：

```
/compact
```

快捷键：`ctrl+x c`

### `/share` — 分享

生成一个链接分享当前对话给团队成员：

```
/share
```

### `/export` — 导出

将当前对话导出为 Markdown 文件：

```
/export
```

快捷键：`ctrl+x x`

### `/themes` — 切换主题

切换 TUI 的配色主题：

```
/themes
```

快捷键：`ctrl+x t`

### `/help` — 帮助

显示帮助信息：

```
/help
```

### `/exit` — 退出

退出 OpenCode：

```
/exit
```

快捷键：`ctrl+x q`

## 核心功能

### 提问与代码解释

你可以像问同事一样向 OpenCode 提问，它会结合整个代码库给出答案。

```
解释一下这个项目中用户认证的完整流程。
```

```
@src/utils/helpers.ts 中的这个工具函数是做什么用的？
```

### 添加新功能

建议三步走：

1. **Plan 模式** — 描述需求，让 AI 出方案

   ```
   当用户删除笔记时，在数据库中标记为已删除而不是真正删除。
   然后创建一个"回收站"页面，用户可以在这里恢复或永久删除笔记。
   ```

2. **迭代方案** — 对方案提出反馈

   ```
   这个回收站页面请参考现有列表页面的风格来设计。
   ```

3. **Build 模式** — 确认方案后让 AI 执行

   ```
   好的，请开始实现。
   ```

### 直接修改代码

对于明确的修改，直接在 Build 模式下提要求：

```
请在 /settings 路由添加登录鉴权，参考 /notes 路由的实现方式。
```

### 撤销修改

不满意 AI 的修改？用 `/undo` 一键回退。

## 实战案例

### 案例一：代码审查

**场景：** 你刚写完一个函数，想让它帮你检查有没有问题。

**操作：**

```diff
- 引用文件：@src/services/user.ts
- 提问：审查这个文件，关注安全性和错误处理。
```

OpenCode 会逐段分析代码，指出潜在问题（如 SQL 注入、未捕获的异常、类型错误）并给出改进建议。

---

### 案例二：给项目加功能

**场景：** 你的博客项目缺少"搜索"功能。

**操作：**

```diff
- 切换到 Plan 模式（按 Tab）
- 提问：我想给这个博客加一个全文搜索功能，支持按标题和内容搜索文章。
```

AI 会分析你的数据模型和路由结构，给出方案。确认后：

```diff
- 切换回 Build 模式（按 Tab）
- 说：好，开始实现吧。
```

AI 会创建搜索路由、编写查询逻辑、添加前端搜索框，一步到位。

---

### 案例三：重构代码

**场景：** 一个文件太长了，想拆分成多个模块。

**操作：**

```diff
- 提问：@src/utils/helpers.ts 这个文件快 500 行了，帮我按功能拆分成多个文件，保持原有导出接口不变。
```

AI 会阅读整个文件，按功能拆分成 `string.ts`、`date.ts`、`validation.ts` 等，并保持各文件的导入导出正确。

---

### 案例四：写单元测试

**场景：** 需要给一个模块写测试。

**操作：**

```diff
- 提问：给 @src/services/user.ts 中的 createUser 和 deleteUser 函数写单元测试。
- 使用项目中已有的测试框架和风格。
```

AI 会查看项目中已有的测试文件，模仿其风格和框架（Jest / Vitest / Mocha），生成测试用例覆盖正常路径和边界情况。

---

### 案例五：修复 Bug

**场景：** 用户报告了一个 Bug，你不知道从哪里下手。

**操作：**

```diff
- 提问：当用户邮箱包含大写字母时登录失败。
- 看一下 @src/services/auth.ts 中登录逻辑，找到问题并修复。
```

AI 会定位到大小写敏感比较的代码行，添加 `.toLowerCase()` 处理，确保大小写不敏感的邮箱匹配。

---

### 案例六：理解遗留代码

**场景：** 你刚接手一个老项目，完全不知道从哪看起。

**操作：**

```diff
- 提问：我刚接手这个项目，给我一个全面的概述：技术栈、目录结构、核心业务流程、关键入口文件。
```

AI 会扫描整个项目，给出技术栈总览、架构图说明、关键文件索引和业务流程描述，让你快速上手。

---

### 案例七：数据库迁移脚本

**场景：** 需要给用户表加一个头像字段。

**操作：**

```diff
- 提问：给用户表添加一个 avatar_url 字段，生成对应的数据库迁移脚本，
- 并更新 User 模型和类型定义。
```

AI 会读取你的 ORM 配置（Prisma / TypeORM / Sequelize），生成迁移文件 + 更新模型定义 + 类型声明。

---

### 案例八：调试 CI 失败

**场景：** CI 流水线挂了，但你不知道原因。

**操作：**

```diff
- !cat CI 日志文件路径
- 提问：CI 失败了，这是日志。帮我分析原因并修复。
```

AI 会分析日志中的错误信息，定位到具体的编译错误、测试失败或配置问题，给出修复方案。
