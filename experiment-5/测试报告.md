## 6. 遗留问题与风险评估

| 问题 | 影响 | 建议 |
|------|------|------|
| 无认证鉴权 | 所有操作视为同一用户 | 第二期添加 JWT |
| 无前端 UI | 只能通过 API 测试 | 后续实验实现 |
| 种子数据仅 15 题 | 演示数据不足 | 后续扩充到 100+ 题 |
| 无性能测试 | 高并发场景未验证 | 上线前做压力测试 |
| 无覆盖率统计 | 无法量化代码覆盖率 | 后续集成 coverage.py |

## 7. 改进建议

### 代码质量
1. 引入 `coverage.py` 工具，设定覆盖率目标 ≥ 70%
2. 添加类型注解检查（mypy），确保类型安全
3. 将 API 路由拆分到独立文件，提高可维护性

### 测试策略
1. 添加性能测试（locust / k6），验证并发场景
2. 添加安全测试（SQL 注入、XSS 检查）
3. 添加前端组件测试（React Testing Library）

### 开发流程
1. 建立 Git Hook，提交前自动运行测试
2. 配置 CI/CD pipeline，每次推送自动运行全量测试
