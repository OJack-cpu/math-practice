# 考研数学刷题系统 — 测试策略文档

## 1. 项目代码分析

### 1.1 项目结构

| 目录/文件 | 作用 |
|-----------|------|
| `backend/main.py` | FastAPI 应用入口，所有 API 路由和服务层逻辑 |
| `backend/models.py` | SQLAlchemy 数据模型定义（4 张核心表） |
| `backend/schemas.py` | Pydantic 请求/响应模型 |
| `backend/database.py` | SQLite 数据库配置和会话管理 |
| `backend/sm2.py` | SM-2 间隔重复算法（纯函数） |
| `backend/seed.py` | 种子数据填充 |
| `backend/requirements.txt` | 项目依赖 |

### 1.2 模块依赖关系

```
sm2.py (无依赖) ← 被 main.py 调用
models.py ← database.py
models.py ← main.py (通过 SQLAlchemy ORM)
seed.py ← models.py (填充种子数据)
main.py → models.py, schemas.py, sm2.py, seed.py, database.py
```

### 1.3 关键数据流

```
用户答题 → POST /api/answer → 判对错 → 保存 AnswerRecord → SM-2 更新 ReviewSchedule
                                      ↓
                                返回 {is_correct, correct_answer, explanation, review_info}
```

## 2. 测试策略

### 2.1 核心模块与测试优先级

| 优先级 | 模块 | 类型 | 理由 |
|:------:|------|------|------|
| P0 | SM-2 算法 (`sm2.py`) | 单元测试 | 核心业务算法，影响复习计划准确性 |
| P0 | 答题 API (`main.py:submit_answer`) | 集成测试 | 核心业务流程，连接判对错 + SM-2 |
| P0 | 题目 API (`main.py:list_questions`) | 集成测试 | 基础数据查询，被所有模块依赖 |
| P1 | 知识点 API | 集成测试 | 基础数据，逻辑简单 |
| P1 | 仪表盘 API | 集成测试 | 数据聚合查询，涉及多表关联 |
| P1 | 复习调度 API | 集成测试 | 依赖 SM-2 正确性 |
| P1 | 错题本 API | 集成测试 | 分组聚合查询 |
| P2 | 薄弱点分析 API | 集成测试 | 综合统计，依赖答题记录 |
| P2 | 正确率趋势 API | 集成测试 | 时间序列数据 |

### 2.2 测试类型选择

| 类型 | 覆盖范围 | 工具 |
|------|---------|------|
| 单元测试 | SM-2 算法（纯函数，无 I/O） | pytest |
| 集成测试 | 所有 API 端点 | pytest + requests |

### 2.3 边界情况与异常场景

| 场景 | 测试点 |
|------|--------|
| 空数据 | 数据库无数据时 API 返回空列表而非报错 |
| 不存在的资源 | 请求不存在的题目 → 404 |
| 边界值 | 答题时间刚好等于平均时间的 0.5 倍/1.5 倍 |
| 非法输入 | 提交不存在的 question_id |
| SM-2 极值 | EF 最低 1.3、repetitions 归零 |

### 2.4 测试执行顺序

1. 单元测试（SM-2 算法）— 快速验证核心逻辑
2. 集成测试（基础 API：知识点、题目）— 验证数据层
3. 集成测试（核心 API：答题、复习）— 验证业务流
4. 集成测试（统计 API：仪表盘、趋势）— 验证聚合查询
