### 问题 1（严重）：CORS 配置不当 — `allow_origins=["*"]` + `allow_credentials=True`

**位置**：`backend/main.py:51-57`

**原始代码**：
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    ...
)
```

**问题描述**：
- CORS 规范禁止 `allow_credentials=True` 与 `allow_origins=["*"]` 同时使用
- 浏览器会拒绝此类响应，导致跨域请求失败
- `"*"` 在生产环境存在安全风险，允许任意来源访问

**修复方案**：
- 通过环境变量 `ALLOWED_ORIGINS` 配置白名单
- 默认值设为本地开发地址，生产环境覆盖为实际域名

**修复代码**：
```python
_cors_origins = os.getenv("ALLOWED_ORIGINS", "http://localhost:5173,http://localhost:3000").split(",")
app.add_middleware(
    CORSMiddleware,
    allow_origins=[o.strip() for o in _cors_origins],
    allow_credentials=True,
    ...
)
```

### 问题 2（性能）：Dashboard API 存在 N+1 查询问题

**位置**：`backend/main.py:313-349` (`get_dashboard` 中的知识点统计部分)

**问题描述**：
- 对 21 个知识点，每个执行 3 次数据库查询（1 次 count + 1 次 filter count + 1 次 avg）
- 总共执行 63 次查询，响应时间随数据量线性增长
- 循环内逐个查询的模式无法利用数据库聚合优化

**修复方案**：
- 使用单次 `GROUP BY` 聚合查询替代循环查询
- 使用 `func.count`、`func.sum(case(...))`、`func.avg` 在一次 SQL 中完成全部统计

**修复前（N+1）**：
```python
for kp in knowledge_points:
    total = db.query(AnswerRecord).join(Question) \
        .filter(Question.knowledge_point_id == kp.id).count()
    correct = ...  # 又一次查询
    avg_time = ...  # 第三次查询
```

**修复后（单次聚合）**：
```python
stats_rows = db.query(
    KnowledgePoint.id, KnowledgePoint.name, KnowledgePoint.category,
    func.count(AnswerRecord.id).label("total_attempts"),
    func.sum(case((AnswerRecord.is_correct == True, 1), else_=0)).label("correct_count"),
    func.avg(AnswerRecord.time_spent).label("avg_time"),
).outerjoin(Question, ...).outerjoin(AnswerRecord, ...) \
 .group_by(KnowledgePoint.id, KnowledgePoint.name, KnowledgePoint.category).all()
```

### 问题 4（中等）：生产环境启用 `reload=True`

**位置**：`backend/main.py:447-448`

**原始代码**：
```python
uvicorn.run("main:app", host="0.0.0.0", port=8000, reload=True)
```

**问题描述**：
- `reload=True` 会启动文件监控进程，消耗额外资源
- 生产环境不需要也不应启用热重载
- 存在安全风险（reload 模式会暴露更多调试信息）

**修复方案**：
- 通过 `ENV` 环境变量控制，仅 `ENV=development` 时启用

```python
_reload = os.getenv("ENV", "").lower() == "development"
uvicorn.run("main:app", host="0.0.0.0", port=8000, reload=_reload)
```

### 问题 5（中等）：数据库配置硬编码 SQLite

**位置**：`backend/database.py:9`

**原始代码**：
```python
SQLALCHEMY_DATABASE_URL = "sqlite:///./math_practice.db"
engine = create_engine(SQLALCHEMY_DATABASE_URL,
    connect_args={"check_same_thread": False}, ...)
```

**问题描述**：
- 数据库连接串硬编码，无法切换到 PostgreSQL
- `check_same_thread=False` 是 SQLite 特有参数，PostgreSQL 不兼容
- 不符合 12-Factor App 的配置管理原则

**修复方案**：
- 通过 `DATABASE_URL` 环境变量读取连接串，默认回退 SQLite
- 根据 URL scheme 区分数据库类型，应用不同的连接参数

```python
SQLALCHEMY_DATABASE_URL = os.getenv("DATABASE_URL", "sqlite:///./math_practice.db")
if SQLALCHEMY_DATABASE_URL.startswith("sqlite"):
    engine = create_engine(SQLALCHEMY_DATABASE_URL,
        connect_args={"check_same_thread": False}, echo=False)
else:
    engine = create_engine(SQLALCHEMY_DATABASE_URL, echo=False)
```
