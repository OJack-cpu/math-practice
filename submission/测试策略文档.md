# 测试策略文档 —— 考研数学刷题系统

## 一、测试目标

确保考研数学刷题系统的核心功能正确、可靠：
- SM-2 间隔重复算法计算正确
- 全部 API 接口正常响应
- 答题判题逻辑准确
- 复习调度状态机正确运行

## 二、测试范围

| 模块 | 测试类型 | 用例数 | 优先级 |
|------|----------|--------|--------|
| SM-2 算法 | 单元测试 | 20 | 🔴 高 |
| API 接口 | 集成测试 | 13 | 🔴 高 |
| 前端组件 | 手动测试 | - | 🟡 中 |
| 数据库模型 | 隐式测试（API 测试覆盖） | - | 🟡 中 |

## 三、测试框架

| 组件 | 工具 | 说明 |
|------|------|------|
| 测试运行器 | pytest 7.x | Python 标准测试框架 |
| HTTP 客户端 | httpx | FastAPI 推荐，支持 async |
| 数据库 | SQLite 内存模式 | 测试隔离，不污染生产数据 |
| 前端测试 | 手动验证 | 因时间限制暂未自动化 |

## 四、单元测试策略（SM-2 算法）

### 测试用例设计原则
- **等价类划分**：6 种质量评分（0~5）
- **边界值测试**：时间边界（avg_time 的 0.5x / 1.5x）
- **状态机测试**：模拟连续多次复习的状态流转

### 测试用例列表

**`test_calculate_quality`（8 个用例）**

| 用例 | 输入（正确/耗时/均值） | 预期输出 | 覆盖场景 |
|------|------------------------|----------|----------|
| correct_fast | True, 10, 60 | 5 | 完美回忆 |
| correct_normal | True, 45, 60 | 4 | 良好 |
| correct_slow | True, 120, 60 | 3 | 费力 |
| wrong_fast | False, 10, 60 | 2 | 粗心错误 |
| wrong_normal | False, 45, 60 | 1 | 不熟悉 |
| wrong_slow | False, 120, 60 | 0 | 完全不会 |
| boundary_avg | True, 60, 60 | 3/4 | 边界值 |
| boundary_half | True, 30, 60 | 4/5 | 边界值 |

**`test_sm2_update`（12 个用例）**

| 用例 | 输入（quality/ef/interval/rep） | 预期输出 | 覆盖场景 |
|------|--------------------------------|----------|----------|
| quality_0_resets | 0, 2.5, 10, 5 | interval=1, rep=0 | 忘记重置 |

| quality_2_resets | 2, 2.5, 10, 5 | interval=1, rep=0 | 犹豫重置 |

## 五、集成测试策略（API 接口）

### 测试设计
- 每个测试独立运行（fixture 自动创建/销毁测试数据库）
- 覆盖正常场景和异常场景（404、参数校验）
- 验证响应状态码和核心字段
