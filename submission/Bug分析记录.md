## BUG-001：LaTeX 数学公式渲染为乱码

**严重程度**：🔴 严重  
**发现时间**：2026-07-02  
**复现步骤**：
1. 打开刷题页面
2. 查看题目内容
3. 数学公式显示为原始 LaTeX 代码（如 `$\lim_{x \to 0}$`）

**根因分析**：
- 前端使用 `<Paragraph>` 直接渲染文本，无 LaTeX 解析能力
- 题目数据使用 LaTeX 语法编写（如 `\frac{}{}`, `\lim`, `\int` 等）

**修复方案**：
1. 安装 KaTeX 库：`npm install katex`
2. 创建 `MathText.jsx` 组件，用正则匹配 `$...$` 和 `$$...$$` 并调用 `katex.renderToString()`
3. 更新所有展示数学内容的页面使用 `<MathText>` 组件

**验证方法**：打开任意题目，确认公式正确渲染为数学符号。

## BUG-002：前端端口无法外部访问

**严重程度**：🔴 严重  
**发现时间**：2026-07-02  
**复现步骤**：
1. 在 CloudStudio 环境中启动 Vite 开发服务器
2. 在浏览器中访问 `http://localhost:5173`
3. 连接失败

**根因分析**：
- Vite 默认监听 `[::1]:5173`（IPv6 本地回环地址）
- CloudStudio 是 Docker 容器环境，localhost 指向容器内部
- CloudStudio 端口面板未自动检测到 Vite 服务

**修复方案**：
1. 使用 `npx vite --host 0.0.0.0 --port 35181` 监听所有网络接口
2. 将前端迁移到 CloudStudio 已有的公开端口 35181
3. 利用 Vite proxy 转发 `/api` 请求到后端 8000 端口

**验证方法**：通过端口面板「查看预览」打开前端页面。

## BUG-003：填空题判题逻辑过于严格

**严重程度**：🟡 中等  
**发现时间**：2026-07-02  
**复现步骤**：
1. 答一道填空题，输入 `1/3`
2. 正确答案为 `1/3`，但判为错误

**根因分析**：
- 判题代码只做 `user_answer.strip().upper() == question.answer.strip().upper()`
- 未处理空格、LaTeX 符号、中文/英文括号、大小写差异

**修复方案**：
1. 新增 `_normalize_answer()` 函数
2. 去除所有空白字符
3. 去除 `$` 和 `\` 符号
4. 统一乘法符号（`×` → `*`）
5. 统一括号（中文 → 英文）
6. 转小写

**修复代码**：
```python
def _normalize_answer(ans: str) -> str:
    ans = ans.strip()
    ans = ans.replace('$', '').replace('\\', '')
    ans = re.sub(r'\s+', '', ans)
    ans = ans.replace('×', '*').replace('x', '*').replace('X', '*')
    ans = ans.replace('（', '(').replace('）', ')')
    return ans.lower()
```

## BUG-004：选项/解析中的 LaTeX 未渲染

**严重程度**：🟡 中等  
**发现时间**：2026-07-01  
**复现步骤**：
1. 答一道选择题
2. 选项中的 LaTeX（如 `A. $\frac{1}{3}$`）显示为源代码

**根因分析**：
- `<MathText>` 仅应用于题目内容，选项和解析仍用纯文本

**修复方案**：
- 选项 `Radio` 内包裹 `<MathText text={opt} />`
- 答案解析的 `correct_answer` 和 `explanation` 用 `<MathText>` 渲染
