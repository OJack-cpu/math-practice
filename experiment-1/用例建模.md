# 用例建模

## 参与者列表

| 参与者 | 类型 | 描述 |
|--------|------|------|
| 学生 | 主要参与者 | 系统核心用户，进行刷题、复习、查看学习数据 |
| 管理员 | 主要参与者 | 负责题库管理和系统维护，同时继承学生的所有权限 |

## 核心用例描述

### UC-001：执行刷题练习

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-001 |
| **用例名称** | 执行刷题练习 |
| **参与者** | 学生 |
| **前置条件** | 1. 用户已登录<br>2. 题库中至少存在一道题目 |
| **基本流程** | 1. 学生进入刷题页面<br>2. 系统显示练习模式选择（顺序练习 / 随机练习 / 错题重练）<br>3. 学生选择练习模式<br>4. 系统根据模式从题库中选取题目<br>5. 系统展示题目，隐藏答案和解析<br>6. 学生阅读题目并作答<br>7. 学生提交答案<br>8. 系统比对答案，显示正确/错误结果<br>9. 系统展示详细解析<br>10. 学生对题目难度进行自评（简单 / 适中 / 困难）<br>11. 学生标记是否掌握该题<br>12. 系统记录答题结果（题目ID、用户答案、正误、答题时长、自评难度、掌握状态）<br>13. 系统更新 SM-2 算法参数<br>14. 系统加载下一道题，重复步骤 5-13<br>15. 所有题目完成后，显示本次练习统计摘要 |
| **扩展流程** | 4a. 所选模式下无可用题目<br>&emsp;4a1. 系统提示"当前模式下无可用题目"<br>&emsp;4a2. 建议用户切换到其他模式或添加题目<br>7a. 学生提交空白答案<br>&emsp;7a1. 系统提示"答案不能为空"<br>&emsp;7a2. 返回步骤 6，等待学生作答<br>8a. 填空题部分正确（多个填空，部分正确）<br>&emsp;8a1. 系统部分标记：正确的空显示绿色，错误的显示红色<br>&emsp;8a2. 整体结果标记为"部分正确"<br>12a. 答题记录写入失败（数据库异常）<br>&emsp;12a1. 系统将答题记录缓存到 localStorage<br>&emsp;12a2. 提示用户"网络异常，数据将在恢复后自动同步"<br>&emsp;12a3. 下次页面加载时检查本地缓存，尝试同步|
| **后置条件** | 答题记录已持久化；SM-2 调度参数已根据结果更新 |

### UC-002：执行间隔复习

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-002 |
| **用例名称** | 执行间隔复习 |
| **参与者** | 学生 |
| **前置条件** | 1. 用户已登录<br>2. SM-2 调度表中存在待复习题目 |
| **基本流程** | 1. 学生进入复习页面<br>2. 系统从 SM-2 调度表中查询今日待复习的题目列表<br>3. 系统按优先级排序：过期天数越多优先级越高，同等过期天数按复习次数升序<br>4. 系统展示第一道待复习题目（仅显示题目，不显示答案）<br>5. 学生尝试回忆题目解法<br>6. 学生点击"查看答案"按钮<br>7. 系统展示正确答案和详细解析<br>8. 学生对回忆质量进行自评：<br>&emsp;- 1 - 完全忘记：完全不记得解法<br>&emsp;- 2 - 困难：记得部分思路，但不确定<br>&emsp;- 3 - 犹豫：知道大概怎么做，细节模糊<br>&emsp;- 4 - 轻松：清楚解法，但有少量迟疑<br>&emsp;- 5 - 完美：完全正确且自信<br>9. 系统根据质量评分更新 SM-2 参数：<br>&emsp;- 质量 < 3（完全忘记/困难）：重置间隔为 1 天，加入明日复习队列<br>&emsp;- 质量 = 3（犹豫）：间隔改为原间隔的一半，加入后续复习队列<br>&emsp;- 质量 >= 4（轻松/完美）：按 SM-2 公式计算新间隔：`I(n+1) = I(n) × EF`，其中 EF 根据质量调整<br>10. 系统加载下一道复习题，重复步骤 4-9<br>11. 所有待复习题目完成后，显示今日复习完成页面<br>12. 系统展示本次复习统计：已复习题数、各质量等级占比、预计下次复习日期 |
| **扩展流程** | 2a. 今日无待复习题目<br>&emsp;2a1. 系统显示提示"恭喜！今天没有需要复习的题目 🎉"<br>&emsp;2a2. 展示下次有复习任务的日期<br>&emsp;2a3. 提供"去做新题"快速入口<br>5a. 学生跳过当前题目<br>&emsp;5a1. 系统将题目标记为"跳过"<br>&emsp;5a2. 不更新 SM-2 参数<br>&emsp;5a3. 该题目将在 1 小时后重新加入今日复习队列<br>9a. 质量评分 = 5（完美回忆）连续 3 次<br>&emsp;9a1. 系统将该题标记为"短期精通"<br>&emsp;9a2. 将该题的复习间隔加倍<br>&emsp;9a3. 该题目在 30 天内不再出现在复习队列中<br>9b. 同一题目连续 3 次评分 < 3<br>&emsp;9b1. 系统建议学生查看该题的知识点解析<br>&emsp;9b2. 将该题加入"困难题目"专项列表<br>10a. 复习过程中浏览器意外关闭<br>&emsp;10a1. 已完成的复习记录已持久化，不受影响<br>&emsp;10a2. 未复习的题目在下一次进入复习页面时重新加载 |
| **后置条件** | SM-2 调度参数已更新；复习记录已持久化 |

### UC-003：查看学习数据分析

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-003 |
| **用例名称** | 查看学习数据分析 |
| **参与者** | 学生 |
| **前置条件** | 用户已登录 |
| **基本流程** | 1. 学生进入数据分析页面<br>2. 系统默认展示"今日概览"面板：<br>&emsp;- 今日做题数、正确数、正确率<br>&emsp;- 今日复习题数<br>&emsp;- 今日学习时长（分钟）<br>&emsp;- 连续学习天数<br>3. 学生选择"正确率趋势"视图<br>4. 系统查询最近 30 天每日正确率数据<br>5. 系统使用 ECharts 渲染折线图：<br>&emsp;- X 轴：日期（最近 30 天）<br>&emsp;- Y 轴：正确率（0%-100%）<br>&emsp;- 显示趋势线（移动平均）<br>6. 学生选择"薄弱知识点"视图<br>7. 系统按知识点聚合所有答题记录，计算每个知识点的综合正确率<br>8. 系统渲染雷达图（所有知识点对比）和柱状图（正确率排序）<br>9. 系统列出薄弱知识点列表（正确率 < 60%），每个知识点显示：<br>&emsp;- 知识点名称<br>&emsp;- 总做题数、正确数、正确率<br>&emsp;- 薄弱等级（轻度/中度/严重）<br>&emsp;- "专项训练"按钮<br>10. 学生选择"学习进度"视图<br>11. 系统按考研数学大纲章节展示完成进度<br>12. 系统渲染进度条列表：<br>&emsp;- 绿色：已完成（该知识点题量完成 80%+）<br>&emsp;- 蓝色：进行中（已做题 20%-80%）<br>&emsp;- 灰色：未开始（已做题 < 20%）<br>13. 学生可切换时间范围（7天/30天/全部）刷新数据 |
| **扩展流程** | 2a. 答题记录不足 10 条<br>&emsp;2a1. 系统显示提示"数据量还不多，继续刷题后将展示更详细的分析"<br>&emsp;2a2. 显示当前已做题数的简单汇总信息<br>&emsp;2a3. 提供"去刷题"引导按钮<br>7a. 某个知识点下答题记录不足 5 条<br>&emsp;7a1. 系统将该知识点标注为"数据不足"<br>&emsp;7a2. 该知识点不纳入薄弱分析<br>&emsp;7a3. 但仍显示在图表中，使用浅色标识<br>9a. 学生点击"专项训练"按钮<br>&emsp;9a1. 系统跳转到刷题页面<br>&emsp;9a2. 自动筛选该知识点下的所有题目<br>&emsp;9a3. 默认使用随机练习模式<br>11a. 大纲章节数据为空（未配置知识点结构）<br>&emsp;11a1. 系统显示"知识点结构尚未配置"<br>&emsp;11a2. 展示基础统计信息代替进度追踪 |
| **后置条件** | 无（数据分析为只读操作） |
